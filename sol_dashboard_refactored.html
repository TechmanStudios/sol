About & How To

What is this? A tiny semantic “manifold” where concepts hold density (ρ) and push flow across links.

How to: Inject a word → watch it spread → adjust Belief (ψ) to open/close pathways → tune Pressure (Π) and Damping to change speed/persistence.

Thermodynamic-inspired flow (intuition)

Aim: treat meaning like a “semantic field” where attention/activation can accumulate, spread, and dissipate over time.
How this demo does it: concepts are nodes, relations are edges, and we move density $\rho$ by pressure differences across edges (a conservative, flow-like update).
What’s captured: denser regions push outward; damping fades signal; the belief/mode field $\psi$ changes conductance (how easily flow moves through links).

Live Metrics

Entropy: normalized Shannon entropy of the density distribution (higher = more spread out).
Flux: how much is moving across edges per step.
Active: how many nodes are currently “lit”.

Controls

Inject Density (Query): adds “mass” (ρ) to a concept node. Bigger injections create stronger pressure gradients and faster propagation.

Belief Field (ψ): biases which links become more conductive. Left privileges tech/logic pathways; right privileges spirit/mythos pathways.

Pressure (Π): higher = denser nodes push harder, so waves spread faster.

Damping: higher = faster fade-out (less persistence). Lower = longer-lasting activation.

What You’ll See

Node size grows with density (ρ).
Edge thickness grows with flux magnitude.
Arrow direction indicates flow direction (flux sign).
Edge brightness reflects how open/conductive the link is under the current belief bias.
Hybrid: SOL + LLMs

What is this? A few practical ways to combine SOL-style state/flow with modern LLMs (ChatGPT/Gemini/Grok-class) in a single workflow.

Why it feels like “thinking”

This is a metaphor for thought as a semantic fluid. When you inject a concept, you add “mass” to that idea; it builds local pressure and spreads along available pathways, like diffusion/flow in a thermodynamic system.

Entropy rises as activation distributes across the network; damping acts like dissipation (signal fades). The belief field doesn’t add energy by itself — it changes conductance, like opening/closing channels that guide where the semantic fluid can move.

How this differs from LLMs

SOL here is a live dynamical system: explicit state evolves over time (ρ on nodes + flux on edges) with interpretable rules (pressure, conductance, damping).
Modern LLMs are trained sequence models: they generate text via next-token prediction from learned parameters, conditioned on prompt + context window (no explicit conserved mass/flux on your graph).
Memory model: SOL’s “memory” is the simulation state. LLM continuity usually requires context, retrieval, tools, or an external memory layer.

Hybrid interaction patterns (3)

1) SOL-as-Router (query steering): Use the activation landscape to select which concepts to retrieve/summarize and which mode to apply; then pass that structured context to the LLM as a compact prompt.
2) LLM-as-Interpreter (state narration): Let SOL run; then ask the LLM to translate the state into language: “What clusters lit up, what pathways opened/closed, what thread is emerging?”
3) LLM-as-Editor (graph growth): Use the LLM to propose new nodes/edges (or reweight edges) from text; SOL becomes the runtime engine that shows how those edits change propagation and stability.
Math & Equations

What is this? This panel shows the exact equations used by the simulation loop.

Equations (as implemented)

State: each node carries density $\rho_i(t)\ge 0$ and a mode/belief scalar $\psi_i(t)$.
(This demo uses a conservative / Darcy-style pressure-driven flux on a graph; $k=1$ for $\psi$.)
Pressure (equation of state):
$$p_i = \Pi(\rho_i) = c\,\log(1+\rho_i)$$
Mode-shaped conductance (strictly positive):
$$w_{ij} = \mathrm{clip}\Big(w^{(0)}_{ij}\,\exp\big(\gamma\,\tfrac{\psi_i+\psi_j}{2}\big),\;w_{\min},\;w_{\max}\Big)$$
Edge flux (Darcy-style):
$$j_{ij} = w_{ij}\,(p_i - p_j)$$
Continuity (graph form):
$$\dot{\rho} + B^\top j = s_{\rho}$$
Circulation (not modeled in this demo):
A divergence-free circulatory component (cycle-space flow) is not represented or visualized here.
This demo uses (implementation note):
We apply edge-wise transfers (finite-volume style) and then damping:
$$\rho_i \leftarrow \rho_i - \tfrac{1}{2}\,j_{ij}\,\Delta t,\quad \rho_j \leftarrow \rho_j + \tfrac{1}{2}\,j_{ij}\,\Delta t$$
$$\rho_i \leftarrow \rho_i\,(1-\kappa\,\Delta t),\quad \rho_i\ge 0$$
For numerical stability in the visualization we smooth flux:
$$j_{ij}\leftarrow (1-\Delta t)\,j_{ij}+\Delta t\,\big(w_{ij}(p_i-p_j)\big)$$
Here $\kappa$ is controlled by the Damping slider; $c$ is controlled by the Pressure slider.

This is a simplified, stable, graph-based model of a larger semantic-flow idea; the equations shown here aim to mimic the same qualitative dynamics at a small, inspectable scale.
